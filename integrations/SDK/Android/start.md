# Установка

Mad SDK — это лёгкий клиент для встраивания рекламы в мобильные приложения. Он позволяет запускать кампании без необходимости реализовывать собственную логику таргетинга, выбора и аналитики.

SDK работает по принципу "тонкого клиента": все решения принимает сервер, а SDK:

- передаёт контекст текущего экрана
- получает кампании и их содержимое в виде JSON или встроенных шаблонов
- активирует показ inline и in-app кампаний
- трекает взаимодействия пользователя (просмотры, клики, и т.д.)
- помогает фиксировать конверсии для аналитики и обучения моделей

Эта документация предназначена для мобильных разработчиков и инженеров, которые интегрируют SDK в приложение.


## Шаг 1. Добавление зависимостей.

Временно SDK поставляется в виде готового к интеграции артефакта.

```kotlin
dependencies {
    implementation(name = "mads", ext = "aar")
}
```

Позже SDK будет поставляться через Maven-репозиторий.

## Шаг 2. Инициализация SDK.

Прежде, чем использовать функционал SDK, интегрирующая сторона должна выполнить его инициализацию после старта приложения.

Для инициализации SDK нужно вызвать метод `Mads#init`:
```kotlin
import ru.tander.mads.Mads

class DemoApplication : Application() {

    override fun onCreate() {
        Mads.init(
            context = this,
            openLinkHandler = ::openLink,
        )
    }

    private fun openLink(link: String) {
        Intent(Intent.ACTION_VIEW, link.toUri())
            .addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
            .let(::startActivity)
    }
}
```

## Шаг 3. Конфигурация SDK.

В процессе работы приложение может задавать и изменять значения переменных, используемых SDK.

```kotlin
import ru.tander.mads.Mads

fun onUserIdUpdated(userId: String) {
    Mads.userId = userId
}
```
