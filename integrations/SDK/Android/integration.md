# Интеграция рекламного SDK

## Шаг 1. Добавление зависимостей.

Временно SDK поставляется в виде готового к интеграции артефакта.

```kotlin
dependencies {
    implementation(name = "mads", ext = "aar")
}
```

Позже SDK будет поставляться через Maven-репозиторий.

## Шаг 2. Инициализация SDK.

Прежде, чем использовать функционал SDK, интегрирующая сторона должна выполнить его инициализацию после старта приложения.

Для инициализации SDK нужно вызвать метод `Mads#init`:
```kotlin
import ru.tander.mads.Mads

class DemoApplication : Application() {

    override fun onCreate() {
        Mads.init(
            context = this,
            openLinkHandler = ::openLink,
        )
    }

    private fun openLink(link: String) {
        Intent(Intent.ACTION_VIEW, link.toUri())
            .addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
            .let(::startActivity)
    }
}
```

## Шаг 3. Конфигурация SDK.

В процессе работы приложение может задавать и изменять значения переменных, используемых SDK.

```kotlin
import ru.tander.mads.Mads

fun onUserIdUpdated(userId: String) {
    Mads.userId = userId
}
```

## Шаг 4. Интеграция рекламных форматов.

После успешной интеграции SDK можно приступать к интеграции форматов рекламы в части UI интегрирующего проекта.

Ниже приведён перечень рекламных форматов и их краткое описание.

| Формат                          | Описание                                                                                                                        |
|---------------------------------|---------------------------------------------------------------------------------------------------------------------------------|
| [In-App](integration-in-app.md) | Формат рекламы, встраиваемый в контент приложения во время естественных пауз, таких как окончание выполнения целевого действия. |
